Effect of SD under the null hypothesis

Apr 22, 2019 by Alex Quent
Tags: Bayesian Stats design analysis rmarkdown
Categories: general

Outset of the problem

I recently discussed generating data for a power/design analysis and the following problem came up. Does the standard deviation (SD) has an influence on the test statistics (t-value, p-value, BF) under the null hypothesis? The noise in the or the SD of the data should have no effect. To understand this, look at Cohen’s D, which is the mean divided by the SD. Under the null hypothesis will always be zero because the mean is zero regardless how big the SD is.

To illustrate this, I ran a short simulation where I generated 100000 data sets each with 100 data points from two zero mean normal distributios: one with SD = 1 and a second with SD = 100. Then, I calculate standard t-tests to get the respective t-values and p-values. Since the original discussion was about a Bayesian design analysis, I also ran the Bayesian version of the t-test.

Simulation code

# This script simulates null data with two different SDs
# Libraries
library(BayesFactor)
library(parallel)
library(assortedRFunctions)

# Functions
fun1 <- function(x){
  return(t.test(x)$statistic)
}

fun2 <- function(x){
  return(t.test(x)$p.value)
}

fun3 <- function(x){
  return(as.numeric(as.vector(ttestBF(x))))
}

# Creating cluster
numCores <- detectCores() - 1
print(paste('Cores used:', numCores))
cluster  <- makeCluster(numCores)
clusterExport(cluster, c('ttestBF',
                         't.test'))

# Parameters
n  <- 10000000
s1 <- 1
s2 <- 100

# Generating the data sets
data1 <- matrix(rnorm(n, 0, s1), ncol = 100)
data2 <- matrix(rnorm(n, 0, s2), ncol = 100)
print('Data created')

# Run simulation
# t-values
print('Start simulation')
startTime <- Sys.time()
t1  <- parApply(cluster, data1, 1, fun1)
t2  <- parApply(cluster, data2, 1, fun1)
time1     <- Sys.time()
print(paste('Finished calculating t-values:', time1))

# p-values
p1  <- parApply(cluster, data1, 1, fun2)
p2  <- parApply(cluster, data2, 1, fun2)
time2     <- Sys.time()
print(paste('Finished calculating p-values:', time2))

# BF
bf1 <- parApply(cluster, data1, 1, fun3)
bf2 <- parApply(cluster, data2, 1, fun3)
time3     <- Sys.time()
print(paste('Finished calculating BFs:', time3))

# Concatenating to data frame
df1  <- data.frame(`p-value` = p1,
                   `t-value` = t1,
                   BF = bf1)

df2  <- data.frame(`p-value` = p2,
                   `t-value` = t2,
                   BF = bf2)

# Stopping analysis
stopCluster(cluster)

# Saving results
save.image(file = datedFileNam('nullDataNoise', '.RData'))

print(time3 - startTime)

Displaying results

# Libraries
library(ggplot2)
library(cowplot)

# Loading data
load("U:/Projects/variousFiles/rStuff/nullDataNoise_20190409_160133.RData")

# Specify quantile range for qq plot
probs <- seq(0, 1, 0.01)

# T-value plots
t1_q <- as.numeric(quantile(t1, probs = probs))
t2_q <- as.numeric(quantile(t2, probs = probs))
t_qq <- ggplot() + 
  geom_point(aes(x = t1_q, y = t2_q)) + 
  labs(y = 'T-values for Data 1', 
       x = 'T-values for Data 2') 

t1_hist <- ggplot() + 
  geom_histogram(aes(x = t1)) + 
  labs(y = 'Count', 
       x = '') +
  coord_cartesian(xlim = c(-5, 5), expand = FALSE)

t2_hist <- ggplot() + 
  geom_histogram(aes(x = t2)) + 
  labs(y = 'Count', 
       x = '') +
  coord_flip()

Conclusion

As can bee seen above in the QQ-plots and the histrograms and as expected, there is basically no difference between the datasets drawn from the two distributions in any of the statistics.
