Running RMarkdown on CBU computer and first pilot session

Nov 13, 2017 by Alex Quent
Tags: pilot study rmarkdown
Categories: general experiments

The issues I am describing here are probably most interesting for people also working on a MRC-CBU computer because MRC-CBU’s specific cyber-security regulations. When I tried to create a new RMarkdown file, I got an error message that the package ‘rprojroot’ could not be found.

These the things that I tried and eventually worked:

  • I installed the package as a source package, because there is no binary version of it. However, I couldn’t because I did not have writing privileges in C:\Program Files\R\R-3.1.3\library. This can be fixed for instance by running RStudio as an administrator or by choosing a different folder.
  • I then installed devtools (install.packages(“devtools”) and the other package (devtools::install_github(“krlmlr/rprojroot”)). After that RMarkdown worked for generating .html files, but not for .pdf.
  • Therefore I downloaded MikTex via a proxy (important for CBU’s firewall) and installed it and everything worked.

Today I also had the first pilot session. Here a my various notes and comments:

  • The number of objects is not too high and the time spend in the environment seems to be sufficient (currently between 30 and 45 seconds).
  • In the final experiment, I should have a mark in the middle of the actual room to ensure that all participants start at the same location in the virtual room. In order to maximise experimental control, I should add the possibility to load the scene (i.e. room) by pressing a key. In future version, the instructions in MATLAB should only be seen at the first time.
  • There could be an issue that objects’ memorability is affected by the similarity/dissimilarity of colours and the lighting. Lighting of the objects as in a real environment heavily depends on the object’s location.
  • Generally, I need to adjust the font size of MATLAB scripts. The questions and answer keys should be labelled.
  • The pilot participant pointed out that there could be an issue of clustering, which means that it is difficult to examine the memory of objects located at the working surface because more objects are located there as opposed to objects on the floor.
  • Do the experiment and change the time and think about changing from Y/N + R/K to R/K/don’t know
  • The toaster and towel were difficult to recognize
